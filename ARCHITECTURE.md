# Drug Inventory System - Architecture & Features Documentation

## System Overview

The Drug Inventory and Supply Chain Tracking System is a comprehensive web application designed for managing pharmaceutical drug inventory and supply chain operations across multiple entities:

- **Headquarters (HQ)**: Central administration and inventory management
- **Hospitals**: Drug consumption tracking and ordering
- **Vendors**: Order fulfillment and shipment tracking

## Technology Stack

### Backend
- **Runtime**: Node.js 18+
- **Framework**: Express.js 4.x
- **Database**: MongoDB 6.0 (Atlas)
- **Authentication**: JWT + bcryptjs
- **Validation**: express-validator
- **API Docs**: Swagger/OpenAPI
- **Security**: Helmet.js, CORS, Rate Limiting
- **Logging**: Morgan
- **Testing**: Jest, Supertest

### Frontend
- **Framework**: React 18
- **Routing**: React Router v6
- **Styling**: Tailwind CSS 3
- **HTTP Client**: Axios
- **Icons**: Lucide React
- **Notifications**: React Hot Toast
- **Charts**: Recharts

### Deployment
- **Containerization**: Docker & Docker Compose
- **Repository**: Git
- **Reverse Proxy**: Nginx
- **Cloud**: AWS, Heroku, or any cloud with Docker support

## Data Models & Relationships

### User Models
- **Headquarters**: Admin users managing inventory
- **Hospital**: Staff ordering and consuming drugs
- **Vendor**: Supply chain partners fulfilling orders

### Business Models
- **StockItem**: Drug inventory with current level and critical limit
- **SupplyOrder**: Purchase orders generated by HQ
- **ShipmentTracking**: Timeline of order status changes
- **ConsumptionLog**: Hospital drug usage records

### Audit Models
- **AuditLog**: All system actions for compliance
- **Notification**: User notifications for critical events

## API Endpoints Summary

### Authentication (Open)
```
POST   /api/auth/register        Register new user
POST   /api/auth/login           User authentication
```

### Stock Management (HQ Only)
```
GET    /api/stock                Get all drugs
POST   /api/stock                Add new drug
PUT    /api/stock/:id            Update drug (inactive becomes critical warning)
DELETE /api/stock/:id            Delete drug (only if inactive)
```

### Consumption (Hospital)
```
POST   /api/consume              Log drug consumption (triggers auto PO if critical)
GET    /api/consume              Get all logs (HQ only)
GET    /api/consume/my           Get my logs (Hospital staff)
```

### Orders
```
GET    /api/orders               Get orders (filtered by role)
PUT    /api/orders/:id/assign    Assign vendor to order (HQ)
PUT    /api/orders/:id/status    Update order status (Vendor)
```

### Shipment Tracking
```
GET    /api/shipment/:orderId    Get complete shipment timeline
```

### Notifications
```
GET    /api/notifications        Get user notifications
PUT    /api/notifications/:id/read Mark as read
```

### Audit (HQ Only)
```
GET    /api/audit                Get audit logs (last 200)
```

## Key Features Explained

### 1. Intelligent Stock Management
- Track drug inventory levels in real-time
- Set critical limits for each drug
- Automatic alerts when stock falls below critical level
- Support for different units (tablets, vials, bottles, etc.)

### 2. Automated Purchase Orders
- HQ creates supply orders when stock is low
- Orders can be assigned to vendors
- Status tracking: Pending > Confirmed > In-Transit > Out-for-Delivery > Delivered
- Automatic stock restoration when delivery is confirmed

### 3. Real-time Consumption Tracking
- Hospitals log drug usage
- Tracks stock level snapshot at time of consumption
- Automatically triggers low-stock alerts and POs
- Historical consumption reports for analysis

### 4. Shipment Tracking
- Timeline-based tracking (insert-only pattern for audit trail)
- Location updates at each stage
- Notes and messages from vendors
- Full history preserved for compliance

### 5. Multi-Role Access Control
- **HQ Admin**: Full system access, inventory management, order assignment, audit logs
- **Hospital Staff**: View inventory, log consumption, view orders, notifications
- **Vendor**: View assigned orders, update shipment status

### 6. Audit & Compliance
- Every action logged with user, timestamp, and description
- Actions tracked: LOGIN, LOGOUT, STOCK_UPDATED, STOCK_ADDED, STOCK_DELETED, PO_CREATED, PO_STATUS_CHANGED, VENDOR_ASSIGNED, CONSUMPTION_LOGGED, DELIVERY_ACKNOWLEDGED
- Export capability for regulatory compliance

### 7. Notifications
- Real-time notifications for critical stock levels
- Order status updates
- Delivery acknowledgments
- Mark as read functionality

## Security Features

- **JWT Authentication**: Stateless token-based auth with expiration
- **Password Security**: bcryptjs hashing with salt rounds
- **Role-Based Access**: RBAC middleware enforcing role restrictions
- **Input Validation**: Server-side validation using express-validator
- **CORS Protection**: Configurable allowed origins
- **Rate Limiting**: 100 requests per 15 minutes per IP
- **HTTP Headers**: Security headers via Helmet.js
- **Audit Trail**: Complete logging for forensic analysis
- **Environment Variables**: Secrets not in code
